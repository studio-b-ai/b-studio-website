§ 03 · Objections, handled

The questions you're actually going to ask.

Q-01
Why should I hire a technology studio that runs a textile company?

Because the textile company is the qualification. We built Bolt to run Heritage Fabrics. We built AcuOps because our own Acumatica instance kept breaking. If our software doesn't survive our operations, it doesn't ship. That filter is what you're buying.

Q-02
Will I get a deck or will I get code?

Code. We don't pitch. The first engagement artifact is a working integration, a configured pipeline, or a running service — not a PowerPoint. If your procurement process requires a deck, we'll write one after we've shipped the thing.

Q-03
What stack do you work in?

Acumatica, HubSpot, Shopify, Railway, Orderful, TypeScript, Python, Postgres, BullMQ. If you're on NetSuite, Sage Intacct, Microsoft Dynamics 365 BC, or Infor CloudSuite, we can talk. We don't pretend to be ERP-agnostic — we pretend about nothing.

Q-04
Can I see a demo environment?

We don't have one. Bolt runs Heritage Fabrics' warehouse. AcuOps runs our own CI/CD. Amplify runs our own outbound. If you want to see the software work, we'll show you the production instance it runs on. That's the demo.

Q-05
Who actually does the work?

Kevin Bibelhausen, Principal. And the rest of the five-person team. No junior-partner handoffs. No offshore bench. You'll hear from the person who's going to write the code.
